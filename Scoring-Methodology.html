01 · Dataset & Scope

From the full FCC registry to a de-duplicated analytical base

The analysis covers US telecom operators registered with the FCC. Starting from the full FCC registry of 7,000+ providers, domains were identified, subdomains enumerated, and OSINT-based scanning performed. All inputs are drawn from publicly available, non-intrusive sources — no authenticated scans were attempted and no access controls were circumvented. 02 · De-duplication Methodology

Multiple scan rows merged into a single distinct entity

Multiple scan rows for the same underlying host were merged into a single distinct entity. A host was considered the same entity as another when any of the following conditions held:

  • Identical hostname — exact string match after lowercase normalisation.
  • Redirect chain — an HTTP 301/302 redirect pointed from one host to another within the same domain.
  • Shared external IP — an identical external IP address shared across multiple subdomain entries.
  • www-prefixed alias — a www alias with no unique CVE or web-application findings versus the bare domain.

When entities were merged, all pillar signal columns retained the MAX value across the merged group — ensuring no risk indicator was lost in consolidation. Result: 37,409 raw host rows collapsed to 28,910 distinct entities (8,499 records merged, a ~22.7% reduction).

03 · Scoring Signals (C-Codes)

26 signals, evaluated per host, aggregated by MAX

Each host is evaluated against 26 binary or tiered signals, grouped into five pillars. Signal values are set per host, then aggregated to the domain (carrier) level using the MAX of all distinct entities under that domain.

05 · Composite Score Formula

A three-step, max-dominant composite

The composite score for each host is computed in three steps. The design ensures a single severely compromised pillar cannot be masked by strong scores in the others.

Step 1 — Weighted score
Weighted = P1×0.30 + P2×0.10 + P3×0.20 + P4×0.15 + P5×0.25
Step 2 — Max-dominant composite
Composite = 0.6 × MaxPillar + 0.4 × Weighted
Where MaxPillar is the highest single pillar score. This ensures a domain with one severely compromised pillar cannot be masked by strong scores elsewhere.
Step 3 — Safety floor & penalty
If any pillar ≥ 75 → Composite ≥ MaxPillar − 10  (safety floor)
Penalty: up to +15 points for compound critical signals. Final host score = min(100, Composite + Penalty).
06 · Carrier Headline Score

Max-based: one critical host makes the carrier critical

The carrier-level (domain-level) headline score is the MAX score across all distinct host entities for that carrier. A single Critical host makes the carrier Critical — there is no averaging at the headline level.

07 · Estimated Breach Impact

Scaled off the de-duped average, not the max headline

Each carrier is assigned a breach-cost estimate that scales with the de-duped average score (not the max headline). The de-duped average better reflects the breadth of exposure across all of a carrier's hosts.

Breach-impact model
Est. Breach Impact = $4,880,000 × (0.5 + DeDupedAvg / 100 × 2.5)
Produces a range of $2.44M (score = 0) to $14.64M (score = 100), rounded to the nearest $10,000.
